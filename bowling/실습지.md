# 볼링 점수판 실습지

이 실습지는 `bowling` 예제를 이용해 배열, 구조체, 상태 전이, 점수 계산 로직을 함께 복습하도록 설계되어 있다.

## 목표

- 볼링 점수 계산 규칙을 코드 관점에서 설명할 수 있다.
- 프레임 진행과 점수 계산이 어떻게 나뉘는지 구분할 수 있다.
- 예제 실행 결과와 규칙 문서를 연결해서 해석할 수 있다.

## 준비

- [볼링 규칙](볼링 규칙.md)을 먼저 읽는다.
- `README.md`의 실행 방법으로 프로그램을 한 번 실행한다.
- 10프레임 처리 방식이 왜 다른지 생각해 본다.

## 실습 순서

1. `board.h`와 `board.c`에서 프레임 상태를 저장하는 구조를 찾는다.
2. 일반 프레임과 10프레임을 처리하는 함수가 왜 나뉘어 있는지 확인한다.
3. 스트라이크, 스페어, 오픈 프레임 입력을 각각 한 번씩 만들어 본다.
4. 누적 점수가 언제 업데이트되는지 출력 결과로 비교한다.
5. 플레이어 이름 입력이나 기록 저장 기능을 추가하려면 어디부터 바꿔야 할지 정리한다.

## 체크 질문

- 스트라이크와 스페어는 다음 투구와 어떤 관계가 있는가
- 10프레임은 왜 일반 프레임과 동일하게 처리할 수 없는가
- 점수판 출력과 실제 점수 계산은 어떤 책임 분리가 필요한가

## 제출물

- 실행 화면 캡처 1장 이상
- 스트라이크, 스페어, 오픈 프레임 예시 입력 3개
- 수정하고 싶은 기능 1개와 구현 아이디어

## 확장 과제

- 플레이어 이름을 실행 시 입력받도록 수정한다.
- 기록 저장 기능을 추가한다.
- 최고 점수나 월별 통계 기능을 연결한다.
