- 메모리 주소에는 1바이트 단위로 번호가 붙어 있다. 1바이트 = 8비트
- 컴퓨터는 다음과 같은 동작을 한다.
  1. 명령어 호출(fetch)
  2. 명령어 해석(decode)
  3. 명령어 실행(execute)
  4. 결과 저장(store)
- CPU의 클럭 속도는 1초에 몇 번의 명령어를 처리할 수 있는지를 나타낸다. 1Hz = 1초에 1회, 1GHz = 1초에 10억 회
- 파이프라인: 명령어의 각 단계가 동시에 처리되는 방식. 예를 들어, 명령어 A가 실행 단계에 있을 때 명령어 B는 해석 단계에 있고, 명령어 C는 호출 단계에 있을 수 있다.
- 명령어 세트: CPU가 이해하고 실행할 수 있는 명령어들의 집합. 예를 들어, x86, ARM 등이 있다.
- 레지스터: CPU 내부에 있는 작은 저장 공간으로, 데이터를 빠르게 읽고 쓸 수 있다. 일반적으로 32비트 또는 64비트 크기를 가진다.
- 메모리 계층: CPU 레지스터 > 캐시 > 메인 메모리 > 보조 저장장치
- 캐시: CPU와 메인 메모리 사이에 위치한 고속 메모리로, 자주 사용되는 데이터를 저장하여 CPU의 접근 속도를 높인다. L1, L2, L3 캐시가 있다.
- 가상 메모리: 운영체제가 물리적 메모리를 효율적으로 관리하기 위해 사용하는 기술로, 프로그램이 사용할 수 있는 메모리 공간을 물리적 메모리보다 크게 보이게 한다.
- ROM(Read-Only Memory): 읽기 전용 메모리로, 컴퓨터가 부팅될 때 필요한 펌웨어나 BIOS 등이 저장되어 있다. 일반적으로 변경할 수 없다.
- RAM(Random Access Memory): 임의 접근 메모리로, 컴퓨터가 실행 중인 프로그램과 데이터를 저장하는 데 사용된다. 전원이 꺼지면 데이터가 사라진다.
- CPU는 명령어를 실행하기 위해 메모리에서 데이터를 읽고 쓰는데, 이 과정에서 메모리 계층이 중요한 역할을 한다. 레지스터는 가장 빠른 접근 속도를 제공하지만 용량이 작고, 캐시는 중간 정도의 속도와 용량을 제공하며, 메인 메모리는 가장 느리지만 가장 큰 용량을 제공한다.

## C++ char 배열 개념 정리

### 1. 변수 스코프 (Scope)

- 변수는 선언된 `{}` 블록 안에서만 존재한다.
- `for (int i = 0; ...)`에서 `i`는 `for`문 밖에서 사용할 수 없다.
- 여러 `for`문에서 같은 변수를 쓰려면 바깥에서 선언해야 한다.

### 2. 배열 인덱스 범위

- `char arr[10]`의 사용 가능한 인덱스는 `0 ~ 9`이다.
- `arr[10]`은 존재하지 않는 칸이므로 범위 초과 버그가 발생한다.

### 3. 널 문자 `\0`

- 문자열의 끝을 알리는 신호다.
- char 배열은 항상 마지막에 `\0`이 필요하다.
- `while (arr[i] != '\0')`는 문자열 끝까지 순회하는 대표적인 패턴이다.

### 4. 배열 초기화

- `char arr[10] = {}`는 모든 칸이 `\0`으로 초기화된다.
- 조건 검사 시에는 복사할 배열을 기준으로 확인해야 한다.

### 5. 문자열 이어붙이기

- 첫 번째 배열 길이를 `i`에 저장한다.
- 두 번째 배열은 `fullname[i + j]` 위치부터 채운다.

```cpp
for (i = 0; i < 10; i++) {
    if (firstname[i] != '\0') {
        fullname[i] = firstname[i];
    } else {
        fullname[i] = '\0';
        break;
    }
}

for (j = 0; j < 10; j++) {
    if (lastname[j] != '\0') {
        fullname[i + j] = lastname[j];
    } else {
        fullname[i + j] = '\0';
        break;
    }
}
```

## `cin.getline` 정리

```cpp
cin.getline(배열이름, 배열크기);
```

- 한 줄 전체를 읽는다.
- 공백도 포함해서 읽는다.
- 마지막에 `\0`를 자동으로 넣는다.

```cpp
char name[20];
cin.getline(name, 20);
```

- `cin >> name;`은 공백 전까지만 읽는다.
- `cin.getline(name, 20);`은 엔터까지 읽는다.
- 앞에서 `cin >>`를 썼으면 `cin.ignore();` 후에 쓰는 것이 안전하다.

```cpp
int age;
char name[20];

cin >> age;
cin.ignore();
cin.getline(name, 20);
```

- 전처리(preprocessing)는 컴파일 전에 `#include`, `#define` 같은 지시문을 먼저 처리하는 단계다.
- 즉, 소스 코드를 바로 번역하는 것이 아니라 번역하기 좋게 먼저 정리하는 과정이다.
- 세상 모든것은 객체이다.
- 의자의 요소:  등받이, 쿠션, 팔걸이, 바퀴, 다리, 스프링, 높낮이 조절 손잡이,  프레임.
- 의자 객체는 이 요소들을 조합해서 만들어진다.
- 객체 지향 프로그래밍에서는 이런 요소들을 클래스의 멤버 변수로 표현한다.
- 클래스는 객체의 설계도와 같다.
- 클래스는 객체의 속성과 행동을 정의한다.
- 객체는 클래스의 인스턴스(instance)로, 클래스에서 정의한 속성과 행동을 실제로 구현한 것이다.
- 클래스는 멤버 변수와 멤버 함수로 구성된다.
- 멤버 변수는 객체의 상태를 나타내는 데이터이다.
- 멤버 함수는 객체의 행동을 나타내는 함수이다.
- 객체 지향 프로그래밍의 주요 특징은 캡슐화(encapsulation), 상속(inheritance), 다형성(polymorphism)이다.

## C++ 클래스 개념 및 코드 구조

### 개념

- **클래스(class)**: 데이터(멤버 변수)와 함수(멤버 함수)를 하나로 묶은 설계도. 클래스 자체는 메모리를 차지하지 않으며, 객체를 생성해야 실제로 사용 가능하다.
- **객체(object)**: 클래스를 바탕으로 실제로 생성된 변수. 클래스 하나로 객체를 여러 개 만들 수 있다.
- **`public`**: 클래스 외부에서 멤버 변수와 함수에 접근할 수 있도록 허용하는 접근 지정자.
- **`::`(범위 지정 연산자)**: 멤버 함수를 클래스 외부에서 구현할 때 어느 클래스에 속하는지 명시한다.

### 코드 구조

```cpp
class 클래스명 {
public:
    // 1. 멤버 변수 선언 (데이터)
    // 2. 멤버 함수 선언 (기능의 목차)
};

반환타입 클래스명 :: 함수명() {
    // 3. 멤버 함수 구현 (기능의 본문)
}

int main() {
    클래스명 객체명;         // 4. 객체 생성
    객체명.멤버변수 = 값;    // 5. 데이터 설정
    객체명.멤버함수();       // 6. 기능 호출
}
```

### 2026-04-10

- 생성자(Constructor): 객체가 생성될 때 자동으로 호출되는 특별한 멤버 함수. 객체의 초기화를 담당한다. 클래스 이름과 동일한 이름을 가지며 반환 타입이 없다.

```cpp
class MyClass {
public:
    MyClass() {
        // 생성자 본문
    }
};
```

- 소멸자 (Destructor): 객체가 소멸될 때 자동으로 호출되는 특별한 멤버 함수. 객체가 사용하던 자원을 해제하는 역할을 한다. 클래스 이름 앞에 `~`를 붙여서 정의한다.

### 2026 -04-13

- 객체 포인터 : 객체의 주소를 저장하는 포인터. 객체의 멤버에 접근할 때 `->` 연산자를 사용한다.

```cpp
MyClass* objPtr = new MyClass(); // 객체 포인터 생성
objPtr->멤버함수(); // 객체 포인터를 통해 멤버 함수 호출
```

- 동적할당 : 런타임에 메모리를 할당하는 방법. `new` 연산자를 사용하여 객체를 동적으로 생성할 수 있다. 동적으로 할당된 객체는 `delete` 연산자를 사용하여 명시적으로 해제해야 한다.

```cpp
MyClass* objPtr = new MyClass(); // 동적 할당
// ... 객체 사용 ...
delete objPtr; // 동적 할당 해제
```

- 소멸자에서 동적할당을 해제하는 이유 : 객체가 소멸될 때 자동으로 호출되는 소멸자에서 동적할당된 메모리를 해제하면, 메모리 누수를 방지할 수 있다. 객체가 더 이상 필요하지 않을 때 메모리를 해제하여 시스템 자원을 효율적으로 관리할 수 있다.

### 2026-04-15

- 참조자(Reference): 변수에 별칭을 붙이는 기능. 참조자는 선언 시 반드시 초기화해야 하며, 이후에는 다른 변수로 참조를 변경할 수 없다.

```cpp
int x = 10;
int& ref = x; // ref는 x의 참조자
ref = 20; // x의 값이 20으로 변경됨
```

- 참조자와 포인터의 차이점: 참조자는 선언 시 반드시 초기화해야 하며, 이후에는 다른 변수로 참조를 변경할 수 없다. 반면 포인터는 선언 후에도 다른 변수의 주소를 가리킬 수 있다.

- 참조자는 포인터보다 사용이 간편하며, 참조자를 통해 멤버에 접근할 때 `.` 연산자를 사용한다.

```cpp
MyClass obj;
MyClass& objRef = obj; // obj의 참조자
objRef.멤버함수(); // 참조자를 통해 멤버 함수 호출
```
