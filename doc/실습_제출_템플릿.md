# 실습 제출 템플릿

---

이 문서는 개인 과제와 팀 프로젝트 제출 시 기본적으로 포함해야 하는 내용을 정리한 템플릿이다.

## 저장소 기본 구조

---

다음 항목을 가능한 한 포함한다.

- `README.md`
- 소스 코드 파일
- 빌드 방법 또는 실행 방법
- 사용한 입력 데이터 또는 테스트 방법
- 발표 자료 또는 결과 이미지

## `README.md` 템플릿

---

```md
# 프로젝트 이름

## 1. 목표
- 무엇을 만드는 프로젝트인지 한 문장으로 설명

## 2. 팀원 또는 작성자
- 이름
- 맡은 역할

## 3. 실행 방법
- 필요한 도구
- 빌드 명령
- 실행 명령

## 4. 주요 기능
- 기능 1
- 기능 2

## 5. 사용한 기술
- C
- CMake
- MQTT / UART / 센서 / GPIO 등

## 6. 어려웠던 점과 해결
- 문제
- 해결 방법
```

## 제출 전 체크리스트

---

- 저장소 첫 화면에서 프로젝트 목적이 보이는가
- 실행 명령이 빠져 있지 않은가
- 팀원 역할이 적혀 있는가
- 하드웨어 연결 또는 네트워크 설정이 필요한 경우 설명이 있는가
- 발표 자료와 저장소 설명이 일치하는가

## 강사용 확인 포인트

---

- 학생이 본인 코드의 흐름을 설명할 수 있는가
- 데모 실패 시 원인을 말할 수 있는가
- 저장소만 보고 재현 가능한가
