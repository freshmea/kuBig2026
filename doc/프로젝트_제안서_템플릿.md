# 프로젝트 제안서 템플릿

---

이 문서는 팀 프로젝트 시작 전에 최소한으로 정리해야 하는 항목을 템플릿 형태로 제공한다.

## 작성 목적

---

- 무엇을 만들지 빠르게 합의한다.
- 구현 범위와 역할 분담을 미리 정한다.
- 발표 때 "왜 이 프로젝트를 했는가"를 설명할 근거를 만든다.

## 템플릿

---

```md
# 프로젝트 제안서

## 1. 프로젝트명
- 팀 이름 또는 프로젝트 이름

## 2. 해결하려는 문제
- 어떤 불편함이나 상황을 해결하려는지 3문장 이내로 설명

## 3. 핵심 기능
- 필수 기능 1
- 필수 기능 2
- 가능하면 넣고 싶은 확장 기능

## 4. 사용할 기술
- MCU: Atmega128 또는 Pico2W
- 통신: MQTT / UART / I2C / SPI 중 해당 항목
- 언어 및 빌드: C, CMake

## 5. 입력과 출력
- 입력 데이터 또는 센서
- 출력 장치 또는 결과 화면

## 6. 팀원 역할
- 이름: 담당 기능
- 이름: 담당 기능

## 7. 일정
- 1차 마일스톤
- 2차 마일스톤
- 발표 준비 일정

## 8. 예상 위험 요소
- 하드웨어 수급
- 센서 연결 실패
- 통신 오류
```

## 작성 기준

---

- 기능 설명보다 문제 정의를 먼저 쓴다.
- "가능하면"과 "반드시"를 구분한다.
- 팀원 역할은 겹쳐도 되지만 책임자는 명확히 둔다.
