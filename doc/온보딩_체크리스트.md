# 온보딩 체크리스트

---

이 문서는 수강생이 첫 수업 전에 완료해야 하는 환경 준비와 첫 실습 진입 순서를 정리한 체크리스트다.

## 수업 전 준비

---

- GitHub 계정을 만들고 저장소 접근 권한을 확인한다.
- Windows 사용자는 WSL2 Ubuntu를 설치한다.
- `git`, `cmake`, `gcc`, `code` 명령이 실행되는지 확인한다.
- VS Code에서 Remote WSL 확장을 설치한다.
- 강의 슬라이드와 공지 채널 링크를 북마크한다.

## 저장소 준비

---

1. 저장소를 clone 한다.
2. 저장소 루트에서 `git submodule update --init --recursive` 를 실행한다.
3. `README.md`를 읽고 현재 주차의 실습 범위를 확인한다.
4. `doc/c_programming.md` 또는 `doc/atmega128.md`에서 오늘 수업 항목을 찾는다.
5. 호스트 예제 실습 전에는 `cmake`, `gcc` 가 설치되어 있는지 다시 확인한다.

## 첫날 완료 기준

---

- `Hello World` 수준의 C 프로그램을 열고 수정할 수 있다.
- VS Code에서 WSL 환경으로 파일을 열 수 있다.
- CMake 기반 프로젝트와 단일 C 파일 프로젝트의 차이를 설명할 수 있다.
- 과제 제출용 개인 저장소 또는 브랜치 전략을 이해한다.

## 자주 막히는 항목

---

- WSL 실행이 안 되면 Windows 기능의 `VirtualMachinePlatform` 설정을 다시 확인한다.
- VS Code가 WSL에 붙지 않으면 Remote WSL 확장과 Ubuntu 기본 배포판 설정을 확인한다.
- `cmake` 또는 `gcc`가 없으면 패키지 설치 여부를 먼저 점검한다.
- USB 장치 실습은 첫 주 C 언어 기초가 끝난 뒤 진행하므로 초반에는 MCU 장치 연결에 시간을 쓰지 않는다.
