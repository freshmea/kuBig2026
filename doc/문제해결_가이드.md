# 문제 해결 가이드

---

수업 중 자주 발생하는 환경 문제와 확인 순서를 정리한 문서다. 강사와 수강생이 같은 순서로 점검하면 디버깅 시간이 크게 줄어든다.

## 1. WSL 또는 VS Code 연결 문제

---

증상:

- WSL 터미널이 실행되지 않는다.
- VS Code에서 Remote WSL 연결이 실패한다.

확인 순서:

1. Windows 기능에서 `Microsoft-Windows-Subsystem-Linux`, `VirtualMachinePlatform`이 켜져 있는지 확인한다.
2. `wsl -l -v`로 Ubuntu 배포판이 설치되어 있고 기본값인지 확인한다.
3. VS Code에서 Remote WSL 확장이 설치되어 있는지 확인한다.
4. WSL 터미널에서 저장소가 정상 경로에 clone 되어 있는지 확인한다.

## 2. `cmake` 또는 `gcc` 명령이 없다

---

증상:

- 빌드 명령 실행 시 command not found가 발생한다.

확인 순서:

1. 패키지 설치 여부를 확인한다.
2. PATH 설정이 꼬였는지 확인한다.
3. 수업 중에는 설치부터 다시 시도하지 말고 옆 사람과 환경 차이를 먼저 비교한다.

## 3. Git clone 또는 push 실패

---

증상:

- 인증 실패
- 원격 저장소 접근 거부

확인 순서:

1. GitHub 로그인 상태와 저장소 권한을 확인한다.
2. `git config --global user.name`, `git config --global user.email` 설정을 확인한다.
3. HTTPS 또는 SSH 중 현재 수업에서 사용하는 방식이 무엇인지 통일한다.

## 4. Atmega128 장치 인식 실패

---

증상:

- USB 장치가 WSL에서 보이지 않는다.
- 업로드 또는 통신 실습이 실패한다.

확인 순서:

1. Windows에서 `usbipd list`로 장치가 보이는지 확인한다.
2. `usbipd bind --busid ...`, `usbipd attach --wsl --busid ...` 순서가 맞는지 확인한다.
3. WSL 안에서 `udev` 규칙이 적용되었는지 확인한다.
4. 케이블 불량과 전원 문제를 먼저 배제한다.

## 5. UART 문자 깨짐 또는 입력 누락

---

증상:

- 3자 이상 입력 시 문자가 깨진다.
- 시리얼 모니터 출력이 불안정하다.

확인 순서:

1. 보드레이트 설정이 송신 측과 수신 측에서 같은지 확인한다.
2. 수신 버퍼 크기와 종료 조건을 점검한다.
3. 인터럽트 처리 중 공유 변수 접근 방식을 다시 확인한다.
4. LCD 출력, UART 출력, 입력 처리 로직을 한 번에 수정하지 말고 하나씩 분리해서 본다.

## 수업 운영 원칙

---

- 환경 문제와 코드 문제를 동시에 잡으려 하지 않는다.
- 재현 절차를 먼저 적고, 그 다음 설정을 바꾼다.
- 장치 실습은 팀 단위로 확인하되, 최종 해결 기록은 각자 남긴다.
