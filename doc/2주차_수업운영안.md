# 2주차 수업 운영안

---

목표는 수강생이 구조체, 파일 입출력, 동적 메모리, 자료구조, MQTT 기초를 연결해서 "작은 프로젝트를 스스로 시작할 수 있는 상태"에 도달하게 만드는 것이다.

## 주간 목표

---

- 구조체와 배열을 함께 사용해 데이터를 표현한다.
- 파일에서 데이터를 읽고 결과를 저장하는 흐름을 이해한다.
- 동적 메모리 할당이 필요한 이유를 설명한다.
- 자료구조 예제와 MQTT 예제를 프로젝트 문맥으로 연결한다.

## 차시 운영

---

### Day 1. 구조체와 문자열

- 핵심 주제: `struct`, `typedef`, 구조체 배열, 문자열 함수
- 추천 예제: `c_example/part5/struct.c`, `structArray.c`, `stringExample.c`
- 실습 산출물: 학생 정보 또는 센서 정보를 구조체 배열로 저장하는 프로그램
- 확인 질문:
  - 구조체 배열이 일반 배열과 다른 점은 무엇인가
  - `strcpy`와 `strncpy`를 왜 구분해야 하는가

### Day 2. 표준 입출력과 파일 처리

- 핵심 주제: `scanf`, `printf`, `fgets`, `fprintf`, 파일 포인터
- 추천 예제: `c_example/part5/charGetPut4.c`, `c_example/part6/filePointer.c`
- 실습 산출물: 파일 입력을 받아 요약 결과를 새 파일에 저장하는 프로그램
- 확인 질문:
  - 표준 입력과 파일 입력은 코드에서 어떻게 닮았고 어떻게 다른가
  - 파일이 열리지 않았을 때 먼저 확인할 것은 무엇인가

### Day 3. 동적 메모리와 `scoreProcess`

- 핵심 주제: `malloc`, `calloc`, `realloc`, 메모리 해제
- 추천 예제: `c_example/part6/dynamicAllocation.c`, `c_example/part6/scoreProcess`
- 실습 산출물: 입력 데이터 수에 따라 자동으로 동작하는 성적 처리 프로그램
- 확인 질문:
  - 고정 길이 배열 대신 동적 메모리를 쓰는 이유는 무엇인가
  - `free`를 빠뜨리면 어떤 문제가 생길 수 있는가

### Day 4. `iotLogReport`와 중간 난이도 과제

- 핵심 주제: 파일 기반 데이터 분석, 포인터 배열 정렬, 보고서 출력
- 추천 예제: `c_example/part6/iotLogReport`
- 실습 산출물: 새 센서 레코드를 추가하고 `report.out` 변화를 설명하는 보고
- 확인 질문:
  - 원본 배열 대신 포인터 배열을 정렬하는 이유는 무엇인가
  - 위험도 순위를 바꾸는 조건은 무엇인가

### Day 5. 자료구조와 MQTT 연결

- 핵심 주제: Stack/Queue 예제 복습, MQTT publish 흐름
- 추천 예제: `ds/stack/myStack3`, `ds/queue/myQueue3`, `network/mqttPub.c`
- 실습 산출물: 자료구조 예제 수정 1개와 MQTT 실행 명령 1개
- 확인 질문:
  - 자료구조 예제는 프로젝트에서 어디에 쓰일 수 있는가
  - MQTT topic과 payload를 프로젝트 요구사항으로 바꾸면 무엇이 달라지는가

## 강사용 체크포인트

---

- Day 2에는 파일 경로와 작업 디렉터리 개념을 반드시 구분해서 설명한다.
- Day 3에는 메모리 할당 실패와 파일 열기 실패를 따로 다룬다.
- Day 4에는 실습 결과를 코드보다 먼저 말로 설명하게 한다.
- Day 5에는 "동작했다"보다 "어디에 적용할 것인가"를 묻는다.

## 추천 과제

---

- `iotLogReport`에 최고 위험 장치 외 추가 요약 정보 넣기
- MQTT message를 센서 형식 문자열로 바꾸기
