# 발표 구성 템플릿

---

이 문서는 팀 프로젝트 발표를 준비할 때 최소한으로 포함해야 하는 흐름을 정리한 템플릿이다.

## 추천 발표 순서

---

### 1. 문제 정의

- 어떤 문제를 해결하려는지 설명한다.
- 왜 이 프로젝트를 선택했는지 말한다.

### 2. 시스템 구성

- 사용한 보드, 센서, 통신 방식을 설명한다.
- 입력, 처리, 출력 흐름을 한 장으로 정리한다.

### 3. 핵심 기능

- 필수 기능 중심으로 2~3개만 고른다.
- 각 기능이 실제로 어떻게 동작하는지 짧게 보여 준다.

### 4. 구현 중 어려웠던 점

- 장치 연결 문제
- 통신 문제
- 코드 구조 문제

### 5. 해결 방법

- 어떤 실험을 했는지
- 최종적으로 무엇을 바꿨는지

### 6. 데모

- 가장 중요한 사용 시나리오를 먼저 보여 준다.
- 실패 가능성이 높은 부분은 대체 시연 자료도 준비한다.

### 7. 회고와 확장 계획

- 지금 상태에서 부족한 점
- 다음에 개선하고 싶은 기능

## 발표 체크리스트

---

- 저장소 README와 발표 내용이 일치하는가
- 팀원 모두가 본인 파트를 설명할 수 있는가
- 데모 장치와 전원이 준비되었는가
- 문제 정의 없이 기능만 나열하고 있지 않은가
