# 숫자 야구 실습지

이 실습지는 `baseballGame` 예제를 이용해 조건문, 반복문, 배열, 함수 분리를 함께 복습하도록 설계되어 있다.

## 목표

- 숫자 야구 판정 규칙을 코드로 설명할 수 있다.
- 입력, 판정, 종료 조건이 각각 어디에 있는지 구분할 수 있다.
- 기존 게임 로직을 바탕으로 작은 기능을 추가할 수 있다.

## 준비

- `README.md`의 실행 방법을 따라 프로그램을 한 번 실행한다.
- `generate_number`, `input_numbers`, `check_result` 함수 역할을 읽는다.

## 실습 순서

1. `generate_number`가 중복 없는 숫자를 만드는 방식을 설명한다.
2. `check_result`에서 `strike`와 `ball`이 어떤 조건으로 증가하는지 확인한다.
3. 잘못된 입력이 들어왔을 때 프로그램이 어떻게 다시 입력을 받는지 확인한다.
4. 게임을 3번 이상 실행하고 시도 횟수를 비교한다.
5. 시도 횟수 제한 또는 힌트 출력 기능을 추가할 방법을 생각한다.

## 체크 질문

- `strike`와 `ball`의 차이는 무엇인가
- 중복 입력을 막지 않으면 게임이 왜 혼란스러워지는가
- 게임 종료 조건은 어디에서 결정되는가

## 제출물

- 게임 실행 화면 캡처 1장 이상
- 함수별 역할을 정리한 짧은 메모
- 추가하고 싶은 기능 1개와 구현 아이디어

## 확장 과제

- 최대 시도 횟수를 추가한다.
- 이전 입력 기록을 출력한다.
- 난이도를 바꿔 4자리 버전으로 확장한다.
