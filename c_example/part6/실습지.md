# part6 파일 처리 실습지

이 실습지는 `part6` 예제를 이용해 파일 입력, 동적 메모리, 실제 데이터 처리 흐름을 연결해서 복습하도록 설계되어 있다.

## 목표

- 파일 포인터와 파일 디스크립터 차이를 말할 수 있다.
- 동적 메모리가 왜 필요한지 예제로 설명할 수 있다.
- `scoreProcess`, `iotLogReport` 같은 중간 난이도 과제로 넘어갈 준비를 마칠 수 있다.

## 준비

- `part6/README.md`의 추천 순서를 먼저 읽는다.
- `filePointer.c`, `dynamicAllocation.c`, `scoreProcess`, `iotLogReport`를 각각 한 번씩 훑어본다.

## 실습 순서

1. `filePointer.c`에서 파일을 열고 닫는 흐름을 정리한다.
2. `dynamicAllocation.c`에서 메모리 할당과 해제 위치를 확인한다.
3. `scoreProcess`에서 학생 수를 자동 계산하는 이유를 설명한다.
4. `iotLogReport`에서 경고 조건과 정렬 결과를 확인한다.
5. 두 프로그램이 어떻게 프로젝트형 과제로 이어지는지 3문장으로 정리한다.

## 체크 질문

- 고정 배열 대신 동적 메모리를 쓰는 가장 큰 이유는 무엇인가
- 파일 입력 기반 프로그램에서 가장 먼저 확인할 오류는 무엇인가
- `scoreProcess`와 `iotLogReport`는 어떤 점에서 비슷하고 다른가

## 제출물

- 파일 처리 흐름을 정리한 짧은 메모
- `scoreProcess` 또는 `iotLogReport` 중 하나를 골라 설명한 3문장
- 다음에 확장해 보고 싶은 기능 1개

## 확장 과제

- `scoreProcess`와 `iotLogReport`를 비교해 공통 함수 구조를 정리한다.
- `filePointer.c`를 입력 파일 경로 인자를 받는 형태로 바꾼다.
- `dynamicAllocation` 예제를 입력 데이터 수와 연결해 본다.
