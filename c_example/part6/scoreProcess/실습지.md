# 성적 처리 실습지

이 실습지는 `scoreProcess` 예제를 이용해 파일 입력, 구조체 배열, 총점 계산, 등수 계산 과정을 복습하도록 설계되어 있다.

## 목표

- 학생 수를 파일에서 자동으로 세는 이유를 설명할 수 있다.
- 구조체 배열에 성적 데이터를 저장하고 합계와 평균을 계산할 수 있다.
- 총점 기준 등수 계산이 어떻게 동작하는지 말할 수 있다.

## 준비

- `score.dat` 파일 형식을 확인한다.
- `README.md`의 실행 방법을 따라 예제를 한 번 실행한다.
- 생성된 `score.out` 파일을 열어 본다.

## 실습 순서

1. `count_students`가 왜 먼저 호출되는지 설명한다.
2. `inputData`가 몇 개의 필드를 읽는지 확인한다.
3. `calculateScore`에서 새 과목을 추가하려면 무엇을 바꿔야 하는지 생각한다.
4. `calculateRank`가 같은 점수를 어떻게 처리하는지 결과로 확인한다.
5. `score.dat`에 학생 2명을 추가하고 `score.out` 변화를 비교한다.

## 체크 질문

- 학생 수를 하드코딩하면 어떤 문제가 생길 수 있는가
- 총점이 같은 학생이 있으면 현재 등수 계산은 어떻게 동작하는가
- 구조체 배열과 포인터 배열을 함께 쓰는 이유는 무엇인가

## 제출물

- 수정한 `score.dat`
- 변경 후 `score.out`
- 등수 계산 로직을 설명하는 3문장 요약

## 확장 과제

- 과학 점수를 추가해 4과목 기준으로 바꾼다.
- 이름순 정렬 결과도 따로 출력한다.
- 과락 기준을 정해 별도 표시를 추가한다.
