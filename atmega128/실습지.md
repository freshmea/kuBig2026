# Atmega128 GPIO/UART 실습지

이 실습지는 `atmega128/src` 예제 중 GPIO, 스위치, UART 흐름을 중심으로 임베디드 기초를 복습하도록 설계되어 있다.

## 목표

- GPIO 출력과 입력이 각각 어떤 레지스터 설정과 연결되는지 설명할 수 있다.
- 스위치 입력, LED 출력, UART 송수신 예제를 연결해서 볼 수 있다.
- 장치 연결 문제와 코드 문제를 나눠서 점검할 수 있다.

## 준비

- `atmega128/README.md`의 추천 학습 순서를 읽는다.
- `led2.c`, `switch1.c`, `uart1.c`, `uart3.c`를 열어 본다.
- [문제 해결 가이드](../doc/문제해결_가이드.md)의 장치 연결 항목을 먼저 확인한다.

## 실습 순서

1. `led2.c`에서 어떤 포트를 출력으로 설정하는지 찾는다.
2. `switch1.c`에서 입력을 읽는 방식과 pull-up/pull-down 개념을 정리한다.
3. `uart1.c`와 `uart3.c`를 비교해 UART 예제가 어떻게 확장되는지 본다.
4. LED, 스위치, UART 중 하나를 골라 "입력-처리-출력" 흐름으로 설명한다.
5. 장치가 동작하지 않을 때 하드웨어 문제와 코드 문제를 어떻게 구분할지 적는다.

## 체크 질문

- GPIO 출력과 입력 설정은 무엇이 다른가
- UART 예제에서 보드레이트와 버퍼 처리가 왜 중요한가
- 임베디드 예제에서 가장 먼저 확인해야 할 하드웨어 요소는 무엇인가

## 제출물

- GPIO 또는 UART 예제 설명 3문장
- 사용한 예제 파일 경로 1개 이상
- 장치 연결 시 가장 조심해야 할 점 2가지

## 확장 과제

- 스위치 입력으로 LED 패턴을 바꾸는 기능을 추가한다.
- UART 수신 데이터를 LCD 출력과 연결하는 흐름을 설계한다.
- 센서 값을 UART로 전송하는 미니 과제를 만든다.
