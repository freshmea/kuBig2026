# 09 긴 작업 진행률 단계별 실습

## 1. 수업 목표

- 오래 걸리는 Python 작업이 UI를 멈추게 만드는 문제를 이해한다.
- `Thread`와 `Lock`으로 백그라운드 작업 상태를 관리한다.
- 진행률은 Python이 계산하고 JS는 표시만 하도록 구성한다.

## 2. 최종 결과물

- 데모 폴더: `long_task_progress`
- 실행: `uv run long_task_progress/main.py`

최종 동작:

1. 사용자가 작업 시작 버튼을 누른다.
2. Python이 백그라운드 스레드를 시작한다.
3. 작업 스레드가 진행률과 메시지를 갱신한다.
4. JS가 `get_progress()`를 호출해 progress bar에 표시한다.
5. 작업 완료 후 버튼이 다시 활성화된다.

---

## 3. 단계별 실습

### 단계 A. 긴 작업을 버튼 클릭에서 바로 실행해 보기

#### A-1. 쉬운 접근

버튼 클릭 시 Python 함수가 오래 걸리는 반복문을 바로 실행하게 만들 수 있다.

#### A-2. 문제점

- 작업이 끝날 때까지 UI가 반응하지 않을 수 있다.
- 사용자는 앱이 멈춘 것으로 느낀다.
- 진행률을 중간에 보여주기 어렵다.

#### A-3. 해결

긴 작업은 별도 스레드에서 실행하고, 화면은 진행 상태만 조회한다.

강의 포인트:
- 계산과 UI 응답성을 분리하는 것이 데스크톱 앱의 핵심이다.

---

### 단계 B. 작업 상태 변수 만들기

#### B-1. 구현

```python
self._running = False
self._progress = 0
self._message = "대기 중"
```

#### B-2. 설명

- `_running`: 작업 중인지 여부
- `_progress`: 0~100 진행률
- `_message`: 사용자에게 보여줄 상태 설명

#### B-3. 문제점

여러 스레드가 같은 변수를 읽고 쓰면 값이 꼬일 수 있다.

#### B-4. 해결

```python
self._lock = Lock()
```

공유 상태 접근은 `with self._lock:` 안에서 처리한다.

---

### 단계 C. 작업 시작 메서드 구현

#### C-1. 구현

```python
def start_task(self) -> dict[str, object]:
    with self._lock:
        if self._running:
            return self.get_progress()
        self._running = True
        self._progress = 0
        self._message = "작업 시작"

    Thread(target=self._run_task, daemon=True).start()
    return self.get_progress()
```

#### C-2. 문제 해결 개념

- 이미 실행 중이면 새 작업을 만들지 않는다.
- 상태 초기화 후 스레드를 시작한다.
- 메서드는 오래 기다리지 않고 바로 반환한다.

강의 포인트:
- 버튼을 여러 번 눌러도 작업이 중복 실행되지 않게 막아야 한다.

---

### 단계 D. 백그라운드 작업 구현

#### D-1. 구현

```python
def _run_task(self) -> None:
    for step in range(1, 11):
        sleep(0.35)
        with self._lock:
            self._progress = step * 10
            self._message = f"{step}/10 단계 처리 중"
```

#### D-2. 설명

이 예제에서는 `sleep()`으로 긴 작업을 흉내 낸다.
실제 수업에서는 파일 변환, 이미지 처리, 대량 계산으로 바꿀 수 있다.

#### D-3. 트릭

스레드 내부에서 UI를 직접 만지지 않는다.
스레드는 Python 상태만 갱신하고, UI는 그 상태를 읽어 표시한다.

---

### 단계 E. 진행률 조회

#### E-1. 구현

```python
def get_progress(self) -> dict[str, object]:
    with self._lock:
        return {
            "running": self._running,
            "progress": self._progress,
            "message": self._message,
        }
```

#### E-2. JS 역할

JS는 200ms 간격으로 조회해서 화면에 표시한다.
이 폴링은 계산용이 아니라 표시용이다.

강의 포인트:
- 핵심 작업은 Python에 있고, JS는 상태 표시만 한다.
