# 11 화면 전환 단계별 실습

## 1. 수업 목표

- 프론트엔드 라우터 없이 작은 데스크톱 앱의 화면 전환을 구현한다.
- 화면에 표시할 데이터는 Python이 제공하고, JS는 현재 화면을 렌더링만 한다.
- 상태 기반 렌더링 개념을 간단한 구조로 이해한다.

## 2. 최종 결과물

- 데모 폴더: `screen_router`
- 실행: `uv run screen_router/main.py`

최종 동작:

1. 앱 시작 시 Python이 화면별 데이터를 반환한다.
2. JS는 버튼 클릭에 따라 현재 화면 이름을 바꾼다.
3. 선택된 화면의 제목과 본문을 표시한다.

---

## 3. 단계별 실습

### 단계 A. HTML 파일을 여러 개 만들지 생각하기

#### A-1. 쉬운 접근

`home.html`, `settings.html`, `detail.html`처럼 파일을 나눌 수 있다.

#### A-2. 문제점

- 화면이 늘어날수록 파일 이동과 상태 전달이 복잡해진다.
- 작은 데스크톱 앱에는 과한 구조일 수 있다.
- 프론트엔드 라우터를 도입하면 JS 학습량이 늘어난다.

#### A-3. 해결

HTML 하나에서 화면 영역만 바꾼다.
화면 데이터는 Python에서 받는다.

강의 포인트:
- Python 심화 수업에서는 복잡한 프론트 라우팅보다 상태 기반 화면 전환이 적합하다.

---

### 단계 B. Python에서 화면 데이터 정의

#### B-1. 구현

```python
class ScreenApi:
    def get_screen_data(self) -> dict[str, object]:
        return {
            "home": {"title": "홈", "body": "상태 기반 화면 전환 데모입니다."},
            "settings": {"title": "설정", "body": "라우터 없이 같은 HTML 안에서 화면만 교체합니다."},
            "detail": {"title": "상세", "body": "표시할 데이터는 Python에서 제공합니다."},
        }
```

#### B-2. 설명

- 화면 이름을 key로 사용한다.
- 각 화면은 제목과 본문을 가진다.
- 실제 앱에서는 Todo 목록, 설정값, 상세 데이터로 확장할 수 있다.

#### B-3. 트릭

화면 데이터를 Python dict로 만들면 나중에 파일, DB, 설정값과 쉽게 연결된다.

---

### 단계 C. JS에서 현재 화면만 선택

#### C-1. 구현

```js
function showScreen(name) {
    const screen = screens[name];
    screenEl.innerHTML = `<h1>${screen.title}</h1><p>${screen.body}</p>`;
}
```

#### C-2. JS 역할

- 버튼 클릭 감지
- 현재 화면 이름 선택
- Python이 준 데이터 렌더링

#### C-3. JS가 하지 않는 일

- 화면 데이터 생성
- 설정값 판단
- 상세 데이터 조회

강의 포인트:
- "화면 전환"은 프론트 기술처럼 보이지만, 어떤 데이터를 보여줄지는 Python 앱 상태와 연결된다.

---

### 단계 D. 앱 시작 시 데이터 불러오기

#### D-1. 구현

```js
window.addEventListener('pywebviewready', async () => {
    screens = await window.pywebview.api.get_screen_data();
    showScreen('home');
});
```

#### D-2. 문제 해결 개념

`pywebviewready` 전에 API를 호출하면 준비되지 않았을 수 있다.
PyWebView API 호출은 준비 이벤트 이후에 한다.
