# 12 앱 패키징 단계별 실습

## 1. 수업 목표

- PyWebView 앱을 실행 파일로 만들 때 필요한 경로 개념을 이해한다.
- 개발 중 실행 환경과 패키징 후 실행 환경의 차이를 확인한다.
- PyInstaller를 사용하기 전 점검해야 할 정보를 Python으로 수집한다.

## 2. 최종 결과물

- 데모 폴더: `packaging_demo`
- 실행: `uv run packaging_demo/main.py`

최종 동작:

1. Python 버전을 표시한다.
2. 현재 실행 파일 경로를 표시한다.
3. 데모 기준 폴더를 표시한다.
4. 패키징 실행 여부(`sys.frozen`)를 표시한다.
5. PyInstaller 예시 명령을 보여준다.

---

## 3. 단계별 실습

### 단계 A. 그냥 실행 파일로 만들면 될까?

#### A-1. 질문

"PyWebView 앱은 Python 파일 하나만 실행 파일로 만들면 끝날까?"

#### A-2. 문제점

- HTML, CSS, JS 정적 파일도 함께 포함해야 한다.
- 개발 중 경로와 패키징 후 경로가 달라질 수 있다.
- 실행 파일 내부에서 리소스 경로를 찾지 못하면 창이 빈 화면이 될 수 있다.

#### A-3. 해결

패키징 전에 Python에서 실행 환경 정보를 확인한다.

강의 포인트:
- 패키징 문제의 상당수는 코드 문제가 아니라 경로 문제다.

---

### 단계 B. 실행 환경 정보 반환

#### B-1. 구현

```python
class PackagingApi:
    def get_environment(self) -> dict[str, str | bool]:
        return {
            "python": sys.version.split()[0],
            "executable": sys.executable,
            "baseDir": str(BASE_DIR),
            "isFrozen": bool(getattr(sys, "frozen", False)),
        }
```

#### B-2. 설명

- `sys.version`: Python 버전
- `sys.executable`: 현재 실행 중인 Python 또는 실행 파일 경로
- `BASE_DIR`: 현재 코드 기준 폴더
- `sys.frozen`: PyInstaller 등으로 패키징되었는지 확인할 때 자주 사용

#### B-3. 트릭

`getattr(sys, "frozen", False)`처럼 기본값을 주면 개발 환경에서도 오류가 나지 않는다.

---

### 단계 C. 정적 파일 경로 이해

#### C-1. 현재 코드

```python
FRONTEND_DIR = BASE_DIR / "frontend"
url=(FRONTEND_DIR / "index.html").as_uri()
```

#### C-2. 문제점

개발 중에는 이 경로가 잘 동작한다.
하지만 실행 파일로 묶을 때 `frontend` 폴더가 포함되지 않으면 HTML을 찾을 수 없다.

#### C-3. 해결 개념

PyInstaller에서는 `--add-data` 또는 spec 파일로 정적 파일을 포함한다.

강의 포인트:
- PyWebView 앱 패키징은 Python 코드뿐 아니라 프론트 리소스 포함이 핵심이다.

---

### 단계 D. PyInstaller 명령 예시

#### D-1. 구현

```python
def get_pyinstaller_command(self) -> str:
    return "pyinstaller --noconfirm --windowed packaging_demo/main.py"
```

#### D-2. 설명

이 명령은 출발점일 뿐이다.
실제 배포에서는 frontend 폴더 포함, 아이콘, 앱 이름, spec 파일 설정이 추가된다.

#### D-3. 수업 운영 팁

처음부터 패키징을 완성하려고 하지 않는다.
먼저 "왜 경로 확인이 필요한가"를 이해시킨 뒤 spec 파일 수업으로 확장한다.
