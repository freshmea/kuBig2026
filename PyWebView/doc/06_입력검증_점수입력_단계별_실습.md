# 06 입력 검증 점수 입력 단계별 실습

## 1. 수업 목표

- 프론트엔드 검증과 백엔드 검증의 역할을 구분한다.
- 중요한 규칙은 Python에서 최종 검증해야 함을 이해한다.
- 성공/실패 결과를 구조화된 dict로 반환한다.

## 2. 최종 결과물

- 데모 폴더: `validation_scores`
- 실행: `uv run validation_scores/main.py`

최종 동작:

1. 사용자가 이름과 점수를 입력한다.
2. JS가 빈 값 정도만 빠르게 확인한다.
3. Python이 이름 길이, 정수 변환, 점수 범위를 검증한다.
4. 오류가 있으면 오류 목록을 반환한다.
5. 성공하면 이름, 점수, 등급을 반환한다.

---

## 3. 단계별 실습

### 단계 A. HTML input만 만들기

#### A-1. 구현

- 이름 입력 input
- 점수 입력 input
- 검증 버튼
- 결과 영역

#### A-2. 문제점

HTML의 `type="number"`만 믿으면 안 된다.
사용자 입력은 언제든 우회되거나 예상과 다르게 들어올 수 있다.

#### A-3. 해결

최종 검증은 Python 메서드에서 수행한다.

강의 포인트:
- 프론트 검증은 친절함이고, 백엔드 검증은 규칙이다.

---

### 단계 B. JS 1차 검증

#### B-1. 구현

```js
function clientErrors() {
    const errors = [];
    if (nameEl.value.trim().length === 0) {
        errors.push('프론트 검증: 이름을 입력하세요.');
    }
    if (scoreEl.value === '') {
        errors.push('프론트 검증: 점수를 입력하세요.');
    }
    return errors;
}
```

#### B-2. 역할

- 빈 입력을 즉시 알려준다.
- 불필요한 Python 호출을 줄인다.

#### B-3. 한계

- 점수 범위, 정수 변환, 등급 계산은 Python에서 처리한다.
- JS 검증은 삭제되거나 우회될 수 있다고 가정한다.

---

### 단계 C. Python 2차 검증

#### C-1. 구현

```python
def validate_score(self, name: str, score: str) -> dict[str, object]:
    errors: list[str] = []
    clean_name = name.strip()
    if len(clean_name) < 2:
        errors.append("이름은 두 글자 이상이어야 합니다.")
```

#### C-2. 점수 변환

```python
try:
    numeric_score = int(score)
except ValueError:
    errors.append("점수는 정수여야 합니다.")
    numeric_score = 0
```

#### C-3. 문제 해결 개념

문자열 입력은 숫자로 변환할 때 실패할 수 있다.
`try/except`로 실패를 정상적인 오류 응답으로 바꾼다.

강의 포인트:
- 예외를 앱 크래시가 아니라 사용자 메시지로 변환하는 것이 실무적인 검증이다.

---

### 단계 D. 구조화된 결과 반환

#### D-1. 실패 응답

```python
return {"ok": False, "errors": errors}
```

#### D-2. 성공 응답

```python
return {"ok": True, "student": {"name": clean_name, "score": numeric_score, "grade": grade}}
```

#### D-3. 장점

- JS가 `ok`만 보고 분기할 수 있다.
- 오류가 여러 개여도 같은 구조로 처리할 수 있다.
- 나중에 CSV 저장, DB 저장으로 확장하기 쉽다.

## 4. 체크리스트

- [ ] 빈 이름은 JS에서 먼저 잡힌다.
- [ ] 한 글자 이름은 Python에서 거절된다.
- [ ] 숫자가 아닌 점수는 Python에서 거절된다.
- [ ] 0~100 밖의 점수는 Python에서 거절된다.
- [ ] 정상 입력은 등급과 함께 표시된다.
