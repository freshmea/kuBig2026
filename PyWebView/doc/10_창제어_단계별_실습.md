# 10 창 제어 단계별 실습

## 1. 수업 목표

- PyWebView가 일반 웹 페이지와 달리 데스크톱 창을 제어할 수 있음을 이해한다.
- 창 크기 변경, 전체화면, 확인창, 새 창 생성을 Python에서 처리한다.
- JS는 어떤 창 제어 명령을 호출할지만 전달한다.

## 2. 최종 결과물

- 데모 폴더: `window_control_demo`
- 실행: `uv run window_control_demo/main.py`

최종 동작:

1. 버튼을 누르면 JS가 Python 메서드를 호출한다.
2. Python이 PyWebView `Window` 객체를 조작한다.
3. 창 크기 변경, 전체화면 전환, 확인창, 새 창 생성이 실행된다.

---

## 3. 단계별 실습

### 단계 A. 웹앱과 데스크톱 앱 차이 질문

#### A-1. 질문

"브라우저 웹 페이지가 내 OS 창 크기를 마음대로 바꿀 수 있을까?"

#### A-2. 문제점

일반 웹 페이지는 보안과 사용자 경험 때문에 OS 창 제어가 제한된다.

#### A-3. 해결

PyWebView에서는 Python이 창 객체를 가지고 있으므로 데스크톱 앱처럼 창을 제어할 수 있다.

강의 포인트:
- PyWebView는 웹 UI를 쓰지만 실행 환경은 데스크톱 앱이다.

---

### 단계 B. Window 객체 보관하기

#### B-1. 구현

```python
class WindowApi:
    def __init__(self) -> None:
        self._window: webview.Window | None = None

    def set_window(self, window: webview.Window) -> None:
        self._window = window
```

#### B-2. 설명

JS에서 호출되는 API 객체가 창을 제어하려면 현재 `Window` 객체를 알아야 한다.

#### B-3. 트릭

API 객체를 먼저 만들고, `create_window()` 결과를 다시 API 객체에 넣는다.

```python
api = WindowApi()
window = webview.create_window(..., js_api=api)
api.set_window(window)
```

---

### 단계 C. 창 크기 변경

#### C-1. 구현

```python
def resize_large(self) -> str:
    if self._window is not None:
        self._window.resize(760, 560)
    return "760 x 560"
```

#### C-2. 문제 해결 개념

`self._window`가 아직 없을 수 있으므로 `None` 체크를 한다.

강의 포인트:
- 데스크톱 API를 다룰 때는 객체 준비 순서를 항상 의식해야 한다.

---

### 단계 D. 전체화면과 확인창

#### D-1. 전체화면

```python
def toggle_fullscreen(self) -> str:
    if self._window is not None:
        self._window.toggle_fullscreen()
    return "fullscreen toggled"
```

#### D-2. 확인창

```python
ok = self._window.create_confirmation_dialog("확인", "창 제어 API 호출이 성공했습니다.")
```

#### D-3. 설명

확인창 결과는 bool로 돌아온다.
Python이 사용자의 선택을 해석해 문자열로 반환한다.

---

### 단계 E. 새 창 열기

#### E-1. 구현

```python
webview.create_window(
    "Child Window",
    html="<h1>새 창</h1><p>Python에서 생성한 창입니다.</p>",
    width=320,
    height=220,
)
```

#### E-2. 주의점

새 창이 많아지면 창 목록 관리가 필요하다.
이번 수업에서는 창 생성 개념만 보여준다.
