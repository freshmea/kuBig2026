# 교육 운영 프레임워크

---

이 저장소는 C 언어, Atmega128A, 네트워크, Pico 기반 실습을 모두 포함하고 있다. 따라서 교육 자료도 단순한 강의 순서 나열보다 "무엇을 할 수 있게 만들 것인가"를 기준으로 정리하는 편이 더 적합하다.

## 운영 원칙

---

### 1. 결과 중심 설계

각 섹션은 설명 중심이 아니라 학습자가 실제로 수행할 수 있어야 하는 결과 기준으로 구성한다.

- C 언어: 기본 문법을 설명할 수 있다.
- C 언어: 작은 프로그램을 직접 작성하고 디버깅할 수 있다.
- Atmega128A: GPIO, UART, ADC, Timer 예제를 빌드하고 수정할 수 있다.
- 프로젝트: 팀 단위로 기능을 구현하고 결과를 발표할 수 있다.

### 2. 실습 중심 진행

현재 저장소의 강점은 예제 코드가 많다는 점이다. 문서도 예제 코드와 연결되는 방식으로 운영하는 것이 좋다.

- `doc/c_programming.md`는 수업 일지 역할
- `doc/atmega128.md`는 임베디드 수업 일지 역할
- 각 파트의 예제 코드는 실습 과제의 기준점 역할
- `README.md`는 전체 과정 안내와 빠른 진입점 역할

### 3. AI 활용은 검증까지 포함

AI 도구는 초안 작성, 설명 보완, 예제 비교에는 도움이 되지만 결과 검증은 사람이 수행해야 한다.

- AI로 코드 설명 초안 작성 가능
- AI로 문서 초안 정리 가능
- 최종 제출 전에는 빌드, 링크, 문서 경로, 오탈자를 직접 검토

## 권장 교육 흐름

---

### 1단계. 기초 도구 적응

- WSL
- VS Code
- Git
- CMake

성과 기준:

- 저장소를 clone 하고 빌드 디렉터리를 준비할 수 있어야 한다.

### 2단계. C 언어 기초

- 변수, 연산자, 조건문, 반복문
- 배열, 함수, 포인터
- 구조체, 파일 입출력, 동적 할당

성과 기준:

- 예제 코드를 읽고 변형 문제를 풀 수 있어야 한다.

### 3단계. 자료구조와 네트워크

- Stack, Queue, List
- MQTT publisher 예제

성과 기준:

- 자료구조 예제를 직접 수정하고 네트워크 예제를 빌드할 수 있어야 한다.

### 4단계. MCU/IoT 실습

- Atmega128A 입출력
- 인터럽트, 타이머, UART, ADC, PWM
- Pico2W 확장 실습

성과 기준:

- 센서/출력 제어 예제를 기반으로 작은 응용 과제를 수행할 수 있어야 한다.

### 5단계. 팀 프로젝트

- 요구사항 정리
- 역할 분담
- 구현
- 발표

성과 기준:

- 팀 산출물과 발표 자료를 함께 제출할 수 있어야 한다.
