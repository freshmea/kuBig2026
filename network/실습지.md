# MQTT 발행 실습지

이 실습지는 `mqttPub.c`를 이용해 MQTT broker, client id, topic, payload 개념을 익히도록 설계되어 있다.

## 목표

- 명령행 인자로 broker, client id, topic, message를 넘길 수 있다.
- publish 실패 원인을 네트워크 설정과 코드 설정으로 나누어 볼 수 있다.
- 프로젝트에서 MQTT를 어디에 연결할지 구체적으로 말할 수 있다.

## 준비

- 로컬 또는 수업용 MQTT broker 주소를 확인한다.
- `network/README.md`의 실행 인자 형식을 읽는다.
- 필요한 라이브러리 설치 여부를 확인한다.

## 실습 순서

1. 기본 인자 없이 실행할 때 어떤 기본값이 쓰이는지 확인한다.
2. broker 주소만 바꿔 실행해 보고 실패 메시지를 읽는다.
3. topic과 message를 바꿔 두 번 이상 전송한다.
4. 센서 데이터처럼 보이도록 payload 형식을 바꿔 본다.
5. 팀 프로젝트에서 이 예제를 어떤 장치 데이터와 연결할지 적는다.

## 체크 질문

- broker 주소와 topic은 각각 무엇을 의미하는가
- client id가 중복되면 어떤 문제가 생길 수 있는가
- 메시지 전송 실패 시 가장 먼저 확인할 설정은 무엇인가

## 제출물

- 사용한 실행 명령 2개 이상
- 성공 또는 실패 결과 화면 캡처
- 팀 프로젝트 적용 아이디어 3문장

## 확장 과제

- subscribe 예제를 추가한다.
- 사용자 입력을 받아 publish 하도록 바꾼다.
- JSON 형식 payload를 만들어 본다.
